## Поток данных при сохранении

```
1. Пользователь создает/изменяет/удаляет предмет
   ↓
2. ItemDatabase.addItem/updateItem/removeItemAt вызывает _saveDatabase()
   ↓
3. ItemDatabaseScreen вызывает StorageService.saveItemDatabase()
   ↓
4. StorageService открывает Box 'item_database'
   ↓
5. Вызывает itemDatabase.toJsonString()
   ↓
6. Сохраняет JSON в Hive с ключом 'items'
   ↓
7. Логирует ✅ СОХРАНЕНО В HIVE!
```

## Поток данных при загрузке

```
1. ItemDatabaseScreen.initState() вызывает _loadDatabase()
   ↓
2. StorageService.loadItemDatabase вызывается
   ↓
3. StorageService открывает Box 'item_database'
   ↓
4. Получает JSON с ключом 'items'
   ↓
5. Вызывает itemDatabase.fromJsonString(json)
   ↓
6. ItemDatabase заполняется предметами из JSON
   ↓
7. setState() обновляет UI
```

## Особенности реализации

✅ **Ленивая загрузка:**
- База загружается только когда открывается ItemDatabaseScreen
- Не замедляет запуск приложения

✅ **Моментальное сохранение:**
- Каждое действие сохраняется сразу в Hive
- Данные не теряются при краше приложения

✅ **Резервирование:**
- Если Hive повредится, база восстановится из defaults
- В коде всегда есть предметы по умолчанию

✅ **Масштабируемость:**
- Система работает с любым количеством предметов
- Сжатие данных через JSON

## 🔍 Отладка сохранения инвентаря

### Архитектура сохранения

```
InventoryScreen добавляет предмет
    ↓
Вызывает widget.onItemChanged?.call() 
    ↓
MainNavigationScreen._saveInventory()
    ↓
    ├─ StorageService.saveInventory() → Hive (локальное)
    └─ FirestoreService.saveInventory() → Firestore (облако)
```

### Когда добавляете предмет, смотрите в логи (Logcat/Debug Console):

Ищите такие сообщения:

```
➕ Добавляю [название] в инвентарь
   ✓ Добавлено в список
   📞 Вызываю callback сохранения
   ✅ Callback вызван
💾 СОХРАНЯЮ ИНВЕНТАРЬ для [character_id] (X предметов)
📋 Предметы: [список имен]
✅ СОХРАНЕНО ЛОКАЛЬНО (Hive)
✅ СОХРАНЕНО В FIRESTORE (облако)
```

### Если видите ошибку:

```
❌ ОШИБКА СОХРАНЕНИЯ: [ошибка]
```

Значит проблема в сохранении - проверьте:
1. Есть ли интернет для Firestore
2. Есть ли права на запись в Firestore (firebase.json)
3. Правильно ли настроена авторизация в Firebase
4. Есть ли character.id (не null)?

### Если callback не вызывается:

```
НЕ ВИДИТЕ сообщений о callback
```

Проверьте:
1. Передается ли `onItemChanged` из MainNavigationScreen в InventoryScreen
2. Передается ли `onItemChanged` из InventoryScreen в ItemDatabaseScreen
3. Вызывается ли `widget.onItemChanged?.call()` в нужных местах
4. Нет ли ошибок в консоли

### Если предметы не сохраняются локально:

```
✅ СОХРАНЕНО ЛОКАЛЬНО (Hive) НЕ ВИДИТЕ
```

Проверьте:
1. Инициализирован ли Hive в main.dart
2. Открыт ли Box для инвентаря
3. Нет ли ошибок при создании JSON

### Если предметы не сохраняются в облаке:

```
⚠️  Не удалось сохранить в Firestore
```

Проверьте:
1. Авторизован ли пользователь (fb.FirebaseAuth.instance.currentUser?.uid != null)
2. Есть ли character.id (не null)
3. Правильно ли настроены правила Firestore
4. Есть ли интернет соединение

### Проверка локально:

1. Откройте приложение
2. Добавьте персонажа
3. Перейдите в инвентарь
4. Нажмите "База" → выберите предмет → "В инвентарь"
5. Посмотрите логи - должны быть все ✅ сообщения
6. Закройте приложение
7. Откройте заново - предмет должен остаться

### Проверка в облаке (Firestore):

1. Откройте Firebase Console
2. Перейдите в Firestore
3. Проверьте: users → [userId] → characters → [charId] → inventory → data
4. Должны быть items с предметами

# 🚀 Быстрый старт - Сохранение инвентаря

## Как это работает теперь

### Поток добавления предмета:
```
1. Пользователь нажимает "База" в инвентаре
2. ItemDatabaseScreen открывается
3. Пользователь выбирает предмет
4. Нажимает "В инвентарь" или кнопку (+)
5. Предмет добавляется в inventory
6. Вызывается widget.onItemChanged?.call()
7. MainNavigationScreen._saveInventory() сохраняет:
   - Локально в Hive
   - В облако Firestore (если авторизован)
```

## Что сохраняется

- ✅ Все предметы в инвентаре
- ✅ Порядок предметов
- ✅ Статус надетых предметов
- ✅ Синхронизируется между устройствами

## Отладка

Если предметы не сохраняются:

1. **Посмотрите в Logcat** - должны быть логи:
   ```
   ➕ Добавляю [название] в инвентарь
   ✅ Callback вызван
   ✅ СОХРАНЕНО ЛОКАЛЬНО (Hive)
   ✅ СОХРАНЕНО В FIRESTORE
   ```

2. **Если нет логов** - callback не вызывается. Проверьте:
   - Передается ли `onItemChanged` в ItemDatabaseScreen
   - Вызывается ли `widget.onItemChanged?.call()`

3. **Если ошибка Firestore** - проверьте:
   - Авторизован ли пользователь
   - Есть ли character.id (не null)
   - Правильно ли настроены правила Firestore

**Подробная отладка:** смотрите `DEBUG_INVENTORY.md`

## Тестирование

### Локальное сохранение
```
1. Откройте приложение
2. Добавьте персонажа
3. Добавьте предмет в инвентарь
4. Закройте приложение
5. Откройте заново
6. ✅ Предмет должен остаться
```

### Облачное сохранение
```
1. Авторизуйтесь с Firebase User
2. Добавьте предмет
3. Откройте Firebase Console → Firestore
4. Проверьте: users/[userId]/characters/[charId]/inventory/data
5. ✅ Должны быть items с предметами
```
